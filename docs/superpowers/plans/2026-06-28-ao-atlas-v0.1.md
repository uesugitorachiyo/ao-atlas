# AO Atlas v0.1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a new public-safe AO Atlas repo that models lightweight AO stack instances, factory-level workgraphs, bounded context packs, Foundry handoff, and readback links.

**Architecture:** AO Atlas is a self-contained Go CLI with typed JSON contracts and local validation. It compiles public-safe manifests and handoff files only; it does not schedule, execute, approve, publish, or call providers.

**Tech Stack:** Go standard library, JSON fixtures/schemas, POSIX shell readiness gate, `jq` for JSON syntax checks.

## Global Constraints

- Apache-2.0 license.
- No copied workflow files from old local folders.
- No private paths, credentials, machine-local data, provider API keys, live push/tag/release/upload/provider calls, or local-only evidence in tracked examples.
- Generated instance state lives outside the public repo by default or under ignored local paths.
- Cross-platform CLI behavior for Ubuntu, macOS, and Windows path strings.
- AO Atlas remains intake/workgraph/context/handoff/readback only.
